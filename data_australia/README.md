# Australia Data Layer

This folder separates source data from app-ready data.

```text
raw/        Original downloaded files or API responses.
processed/  Cleaned CSV files used by the local agents.
```

## Current Dataset

`processed/community_profiles.csv` is generated by:

```powershell
.\.venv\Scripts\python.exe scripts\download_abs_community_profiles.py
```

The downloader queries the official ABS Data by Region / Digital Atlas of Australia SA2 population and people layer, applies the configured mappings in `region_mappings.yml`, then writes a compact community profile table for the prototype.

It also downloads an ABS ASGS 2021 SA2 boundary GeoJSON subset:

```text
raw/abs_sa2_boundaries_subset.geojson
processed/sa2_coverage.geojson
```

The Streamlit app uses `processed/sa2_coverage.geojson` for the local map view.

## All-Australia SA2 Selection Dataset

For the full Australia map selector, run:

```powershell
.\.venv\Scripts\python.exe scripts\download_abs_sa2_all.py
```

This generates:

```text
raw/abs_population_people_sa2_all.json
raw/abs_sa2_boundaries_all.geojson
processed/sa2_profiles_all.csv
processed/sa2_boundaries_all.geojson
processed/sa2_boundaries_by_state/*.geojson
```

The app can then filter by state/territory, SA4, SA3, SA2, and keyword search.
For better UI performance, the map reads the state-specific GeoJSON files first and only falls back to the full all-Australia file if needed.

## ABS ASGS Allocation and Correspondence Files

For official geography traceability, run:

```powershell
.\.venv\Scripts\python.exe scripts\download_abs_asgs_allocations.py
```

This downloads stable ABS ASGS Edition 3 files and generates:

```text
raw/asgs_allocations/SA2_2021_AUST.xlsx
raw/asgs_allocations/LGA_2025_AUST.xlsx
raw/asgs_allocations/CG_LGA_2024_LGA_2025.csv
processed/asgs_allocations/sa2_to_sa3_sa4_state_2021.csv
processed/asgs_allocations/lga_2025_summary.csv
processed/asgs_allocations/lga_2024_to_2025_correspondence.csv
processed/asgs_allocations/metadata.json
```

These files support official SA2/SA3/SA4/State hierarchy checks, LGA 2025 reference context, and audit traceability. They do not provide live incidents, warnings, evacuation routes or current fire danger.

Source:

- ABS Data by Region: https://dbr.abs.gov.au/
- ABS Digital Atlas ArcGIS layer: https://geo.abs.gov.au/arcgis/rest/services/Hosted/ABS_Population_and_people_by_2021_SA2_Nov_2023/FeatureServer/1
- ABS ASGS Edition 3 allocation files: https://www.abs.gov.au/statistics/standards/australian-statistical-geography-standard-asgs-edition-3/jul2021-jun2026/access-and-downloads/allocation-files
- ABS ASGS Edition 3 correspondences: https://www.abs.gov.au/statistics/standards/australian-statistical-geography-standard-asgs-edition-3/jul2021-jun2026/access-and-downloads/correspondences

## GitHub Publishing Boundary

The repository is set up so that lightweight processed reference files can support a reproducible demo, while large raw downloads and map files remain local:

```text
Committed:
processed/community_profiles.csv
processed/asgs_allocations/*.csv
processed/asgs_allocations/metadata.json

Ignored:
raw/
processed/sa2_profiles_all.csv
processed/sa2_coverage.geojson
processed/sa2_boundaries_all.geojson
processed/sa2_boundaries_by_state/
```

Before commercial use, review the licence register, source terms, attribution requirements, caching rules and redistribution permissions.

## Important Limitations

- The current processed file uses population, older-person counts, and language-at-home fields from the ABS layer.
- Cairns currently uses a configured SA2 approximation. It is stronger than keyword matching, but it should still be verified against an official ABS SA2-to-LGA correspondence before operational use.
- LGA allocation is mesh-block based. It strengthens official geography traceability, but it is not a substitute for human GIS review when SA2 and LGA boundaries do not align cleanly.
- `no_car_households_pct` is intentionally left blank because the current population-and-people layer does not provide that transport variable.
- Operational use should verify current warnings, evacuation orders, local transport conditions, and official emergency instructions separately.
